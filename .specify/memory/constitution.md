# Physical AI Docs Constitution

## Core Principles

### I. Project Scope
All generated content belongs only to **Physical AI** and **Humanoid Robotics**, prepared for a **Docusaurus frontend**.

### II. Content Rules
- Write everything in **English**.
- Style must be **research-focused**, **clean**, and **well-structured**.
- Use proper **Docusaurus markdown** (H1, H2, H3).
- Minimum sections: **Introduction, Syllabus, 6 Chapters, Labs, Research Papers, MCP Servers (7 servers)**.
- No motivational or casual tone.

### III. Tool Rules
Define **7 tools** only as *rules*, not functional features:
1. Analysis
2. Rewrite
3. Structure
4. Chapter Builder
5. Research Formatter
6. Code Block
7. Docusaurus Formatter

❗ **No tool can use web-search or external data.**

### IV. Forbidden
- No web search
- No external queries
- No unrelated topics
- No unsafe robotics instructions

### V. Output Rules
- Output must stay **on-topic** within Physical AI.
- Must follow **Docusaurus-compatible markdown**.
- Docs should support clean UI with **header, footer, hero section**.

## Governance
- This Constitution supersedes all other practices.
- Amendments require documentation, approval, and a migration plan.
- All development activities must verify compliance with these principles.
