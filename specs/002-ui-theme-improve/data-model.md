# Data Model: Improve UI with Physical AI/Robotics Theme

**Date**: December 7, 2025
**Feature**: specs/002-ui-theme-improve/spec.md
**Plan**: specs/002-ui-theme-improve/plan.md

## Entities

### 1. Homepage (`index.tsx`)

*   **Description**: Represents the main landing page of the Docusaurus site (`my-website/src/pages/index.tsx`). This entity's visual presentation will be significantly enhanced.
*   **Attributes**:
    *   `file_path` (string): `my-website/src/pages/index.tsx`
    *   `components` (list of React components): Includes the `HomepageFeatures` component and potentially other UI elements (e.g., Hero section, layout wrappers).
    *   `styling_directives` (string/object): Inline styles or class names applied to elements, referencing custom CSS.
*   **Validation Rules**:
    *   Must render correctly within Docusaurus.
    *   Styling must be driven by `custom.css`.
    *   Must achieve "premium, VIP-level UI" matching Physical AI/Robotics theme.

### 2. HomepageFeatures Component (`index.tsx`)

*   **Description**: A reusable React component (`my-website/src/components/HomepageFeatures/index.tsx`) typically used on the homepage to display key features with icons/titles/descriptions. Its styling will be updated.
*   **Attributes**:
    *   `file_path` (string): `my-website/src/components/HomepageFeatures/index.tsx`
    *   `props` (object): Data passed to the component (e.g., feature data).
    *   `styling_directives` (string/object): Class names or inline styles applied to elements, referencing custom CSS.
*   **Validation Rules**:
    *   Must render correctly as a React component.
    *   Styling must be driven by `custom.css`.
    *   Must exhibit "premium, futuristic design principles."

### 3. Custom CSS (`custom.css`)

*   **Description**: The central stylesheet (`my-website/src/css/custom.css`) where all new custom UI enhancements for the homepage, components, and documentation markdown pages will be defined.
*   **Attributes**:
    *   `file_path` (string): `my-website/src/css/custom.css`
    *   `styles` (CSS rules): Definitions for colors, typography, animations, gradients, neon highlights, layout adjustments.
*   **Validation Rules**:
    *   Must be valid CSS.
    *   Must not contain TailwindCSS syntax or imports of external CSS frameworks.
    *   Must successfully override/augment default Docusaurus styles to achieve the desired aesthetic.

### 4. Documentation Markdown Pages (`docs/*.md`)

*   **Description**: All existing and future markdown files within the `my-website/docs` folder. These pages will receive new styling via `custom.css` for their content (headings, text, tables, diagrams).
*   **Attributes**:
    *   `file_path` (string): A pattern representing all files like `my-website/docs/*.md`
    *   `rendered_html` (dynamic): The HTML output generated by Docusaurus from the markdown, which CSS will target.
*   **Validation Rules**:
    *   Must correctly display content according to markdown syntax.
    *   Must reflect high-quality, premium Physical AI theme styling through `custom.css`.
    *   Headings, tables, diagrams must be cleanly styled.

## Relationships

*   **Homepage** and **HomepageFeatures Component** directly utilize styles defined in **Custom CSS**.
*   **Documentation Markdown Pages** are styled by **Custom CSS** when rendered by Docusaurus.
