# Data Model: Upgrade UI to Premium Physical AI/Robotics Theme with Defined Palette

**Date**: December 7, 2025
**Feature**: specs/003-ui-upgrade-theme/spec.md
**Plan**: specs/003-ui-upgrade-theme/plan.md

## Entities

### 1. Homepage (`index.tsx`)

*   **Description**: Represents the main landing page of the Docusaurus site (`my-website/src/pages/index.tsx`). This entity's visual presentation will be significantly enhanced according to the specified premium theme.
*   **Attributes**:
    *   `file_path` (string): `my-website/src/pages/index.tsx`
    *   `components` (list of React components): Includes the `HomepageFeatures` component and other UI elements (e.g., Hero section, layout wrappers).
    *   `styling_directives` (string/object): Class names or inline styles applied to elements, referencing custom CSS.
*   **Validation Rules**:
    *   Must render correctly within Docusaurus.
    *   Styling must be driven by `my-website/src/css/custom.css`.
    *   Must achieve "premium, VIP-level UI" matching Physical AI/Robotics theme, using the specified color palette.

### 2. HomepageFeatures Component (`index.tsx`)

*   **Description**: A reusable React component (`my-website/src/components/HomepageFeatures/index.tsx`) typically used on the homepage to display key features. Its styling will be updated to align with the premium theme.
*   **Attributes**:
    *   `file_path` (string): `my-website/src/components/HomepageFeatures/index.tsx`
    *   `props` (object): Data passed to the component (e.g., feature data).
    *   `styling_directives` (string/object): Class names or inline styles applied to elements, referencing custom CSS.
*   **Validation Rules**:
    *   Must render correctly as a React component.
    *   Styling must be driven by `my-website/src/css/custom.css`.
    *   Must exhibit "premium, futuristic design principles" and use the specified color palette.

### 3. Custom CSS (`custom.css`)

*   **Description**: The central stylesheet (`my-website/src/css/custom.css`) where all new custom UI enhancements for the homepage, components, and documentation markdown pages will be defined.
*   **Attributes**:
    *   `file_path` (string): `my-website/src/css/custom.css`
    *   `styles` (CSS rules): Definitions for the specified color palette, typography (futuristic, clean fonts), animations (smooth hover effects, light animations), gradients, subtle patterns, spacing, font sizes, and visual hierarchy.
*   **Validation Rules**:
    *   Must be valid CSS.
    *   Must strictly use the specified color palette: `#0A0F1C`, `#FF7A00`, `#4BA3FF`, `#E3E6EB`.
    *   Must not contain TailwindCSS syntax or imports of other external CSS frameworks.
    *   Must successfully override/augment default Docusaurus styles to achieve the desired aesthetic without breaking existing functionality.

### 4. Documentation Layout

*   **Description**: The overall visual presentation and styling of pages within the `my-website/docs` folder. This refers to the application of the premium theme to the rendered markdown content and surrounding layout elements (e.g., header, footer, sidebar).
*   **Attributes**:
    *   `target_scope` (string): All rendered markdown pages within `my-website/docs`.
    *   `rendered_html` (dynamic): The HTML output generated by Docusaurus from the markdown, which CSS will target.
*   **Validation Rules**:
    *   Must display content correctly according to markdown syntax.
    *   Must reflect the high-quality, premium Physical AI theme styling through `my-website/src/css/custom.css`.
    *   Headings, tables, diagrams, and overall page structure must be cleanly styled with enhanced spacing, font sizes, and visual hierarchy.
    *   Existing content must not be deleted or removed.

## Relationships

*   **Homepage** and **HomepageFeatures Component** directly utilize styles defined in **Custom CSS**.
*   **Documentation Layout** is styled by **Custom CSS** when rendered by Docusaurus, affecting elements like headings and markdown content.
